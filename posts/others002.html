🛢️ MySQL 筆記 for beginner

張家銘 · 2025/08/05

你一定常常聽到資料庫這個詞，現代網站和很多遊戲、軟體等等 後面都會有資料庫在管理各式各樣的資料。可以說是非常重要的一個東西，如果學會使用資料庫也可以大大增加自己的就業機會!因為很多各行各業都會需要用到資料庫來統整資料。而今天這篇文章可以幫助大家在看完影片後，用文章做複習和實作!

🚀 準備工作：安裝 & 開啟資料庫

  • 下載並安裝 MySQL 或用 XAMPP / MAMP 裡的 MySQL
  • 用 Terminal 或 MySQL Workbench 連線，輸入帳號密碼登入
  • 第一次使用記得先用：CREATE DATABASE my_database;
  • 然後切換到資料庫：USE my_database;

🏗️ 建表：從零開始設計欄位

表格就像 Excel 格子，每行代表一筆資料。


-- 切換資料庫
USE my_database;

-- 建表語法範例
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    balance DECIMAL(10,2) DEFAULT 0.00,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

語法說明：

  • USE my_database;：切換到名為 my_database 的資料庫，之後的操作都會在這個 DB 裡進行。
  • CREATE TABLE users (...)：建立一個叫 users 的表格。
  • id INT PRIMARY KEY AUTO_INCREMENT：建立整數型的 id 欄位，自動遞增，每筆資料會有獨立的流水號（當主鍵）。
  • name VARCHAR(100) NOT NULL：名稱欄位，最多 100 個字，不能是空值。
  • balance DECIMAL(10,2)：金額數字，最多 10 位數，其中 2 位是小數。
  • created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP：自動加上目前新增時間。

建立表格：

  • id：主鍵唯一id 這裡直接用自動 +1，確保id一定不重複；
  • name：文字欄位，長度最多 100，不能留白；
  • balance：金額格式，兩位小數；
  • created_at：紀錄新增時間。

📥 插入資料：怎麼寫資料進表裡？


INSERT INTO users (name, balance)
VALUES ('Alice', 100.00), ('Bob', 50.50);

語法說明：

  • INSERT INTO users (name, balance)：要插入的欄位是 name 跟 balance，id 和 created_at 系統會自動補。
  • VALUES ('Alice', 100.00), ('Bob', 50.50);：新增一筆資料，name 是 Alice，balance 是 100 元 以及Bob，balance 是 50.50 元。

✔️ 小提醒：INSERT 寫欄位名稱才不會搞錯順序；一次寫多筆用逗號隔開更有效率。

🔄 更新 & 刪除：修改表格中的內容


-- 更新指定資料
UPDATE users
SET balance = balance + 20.00
WHERE name = 'Alice';

-- 刪除資料
DELETE FROM users
WHERE name = 'Bob';

語法說明：

  • UPDATE users：修改 users 表裡的資料。
  • SET balance = balance + 20：將 balance 加 20（目前值 +20）。
  • WHERE name = 'Alice'：只針對 name 是 Alice 的那一筆資料。
  • DELETE FROM users：從 users 表中刪除資料。
  • WHERE name = 'Bob'：限定只刪除名字是 Bob 的資料。

✅ 記得加上 WHERE，避免你一不小心連整張表都清掉…

🔍 查詢資料：SELECT 的基本玩法


-- 選全部資料
SELECT * FROM users;

-- 查特定欄位
SELECT name, balance FROM users;

-- 條件查找
SELECT * FROM users WHERE balance >= 50.00;

-- 排序
SELECT * FROM users ORDER BY balance DESC;

-- 模糊比對
SELECT * FROM users WHERE name LIKE '%li%';

語法說明：

  • SELECT *：選取所有欄位。
  • FROM users：從 users 這張表格裡找資料。
  • SELECT name, balance：只取出 name 和 balance 這兩個欄位。
  • WHERE balance >= 50.00：篩選條件，balance 要大於等於 50.00 才會出現。
  • AND / OR：組合多個條件（AND 全都要符合，OR 任一符合即可）。
  • ORDER BY balance DESC：根據 balance 欄位做排序，DESC 是遞減（大到小），ASC 是遞增（小到大）。
  • LIKE '%li%'：模糊查詢，找出 name 裡「包含 li 」的資料。(%就代表隨便字 例如%le 就會包含apple ap% 也會包含apple，也可以用_代表一個字 例如a_p% 包含apple)

📊 統計 & 分組：COUNT、GROUP BY 用起來


SELECT name, COUNT(*) AS order_count
FROM orders
GROUP BY name;

語法說明：

  • SELECT name, COUNT(*) AS order_count：選出 name 欄位，並用 COUNT(*) 統計每個名字出現幾次（筆數），命名為 order_count(通常會直接顯示在下方，如果你用MySQL Workbench。)。
  • FROM orders：資料來源是 orders 這張表。
  • GROUP BY name：以 name 欄位來分組，也就是每個 name 為一組，然後分別計算每組的筆數。

這樣就可以看每位使用者訂單有幾筆。

🔗 多表關聯：JOIN 和 UNION 怎麼用


-- UNION 查兩張表
SELECT email FROM users
UNION
SELECT email FROM admins;

-- JOIN 常見用法
SELECT orders.id, users.name, orders.total
FROM orders
JOIN users ON orders.user_id = users.id;

語法說明：

  • UNION：把兩個查詢結果上下合併（不重複）。需要查詢欄位數量與型別一致。
  • SELECT name FROM users：從 users 表選出 name。
  • JOIN：把兩張表根據關聯欄位合併在一起，資料橫向擴展。
  • JOIN users ON orders.user_id = users.id：把 orders 裡的 user_id 對應到 users 表的 id。

JOIN 用來把兩張表連起來，務必確保 FK 跟 PK 型態一致。

⚠️ 常見錯誤＆注意事項

  • 名稱別用 SQL 保留字（像 table, select），會出事
  • 表格和欄位沒有強制規定大小寫，但邏輯要統一比較不會搞混
  • 每張表記得要有有一個唯一主鍵
  • 執行 UPDATE 和 DELETE 時務必加 WHERE 條件
  • 如果資料超過欄位長度（如 VARCHAR(10) 你插入11個字），插入時超過的會被截斷!

📝 建議給初學者的小提醒

  • 剛開始熟練 SELECT/INSERT/UPDATE/DELETE 這四個語法
  • 練習題目可以從簡單表格開始，多動手查資料才會記得
  • 試著寫出自己的小資料庫，比如學生名單、成績表，做完會很有成就感！
  • 實作中遇到錯誤，就把錯誤訊息貼 google，說不定一堆人之前也遇過
  • 當你能順利的把資料庫運用自如，你就得到超多工作機會了 💼
